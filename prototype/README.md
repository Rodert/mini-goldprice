# 贵金属回收小程序 - 原型图说明

## 📱 原型图概述

这是一个**贵金属回收店铺微信小程序**的HTML原型图，包含4个核心页面。主要用于展示店铺的回购/销售价格，提供在线预约和价格计算服务。

## 🎯 业务场景

### 适用对象
- 黄金回收店
- 典当行
- 贵金属交易商
- 珠宝回收门店

### 核心业务
- 展示店铺的回购价格和销售价格
- 提供在线预约回收服务
- 帮助客户快速估算贵金属价值
- 管理客户预约和回收记录

### 2. 预约回收 (appointment.html)

**功能说明**：
- 用户可以预约到店或上门回收服务
- 选择回收品种、预约时间
- 填写联系信息
- 查看预约记录

**表单字段**：
- 回收品种：黄金/白银/铂金/钯金（必填）
- 预约方式：到店回收/上门回收（必填）
- 预约时间：日期 + 时间段（必填）
- 您的姓名（必填）
- 联系电话（必填）
- 详细地址（上门回收必填）
- 备注说明（选填）

**服务说明**：
- 预约后工作人员会在30分钟内联系确认
- 到店回收无需任何费用，上门回收视情况收取服务费
- 请携带有效身份证件及物品相关票据
- 预约时间可提前2小时取消或改期

**预约记录**：
- 显示预约状态（已确认/待确认）
- 预约时间、方式、联系方式
- 支持查看和管理历史预约

### 3. 价格计算器 (calculator.html)

**功能说明**：
- 帮助用户快速估算贵金属回收价值
- 支持多品种、多成色计算
- 显示计算明细和历史记录

**计算步骤**：
1. 选择贵金属品种（黄金/白银/铂金/钯金）
2. 系统显示当前回购价格
3. 输入重量（克）
4. 输入成色系数（可选，默认99.9%）
5. 点击计算按钮
6. 显示预估回收价值

**计算公式**：
```
回收价值 = 重量(克) × 回购价(元/克) × 成色系数(%)
```

**示例**：
```
品种：黄金 Au9999
回购价：¥558.50/克
重量：50克
成色：99.9%
结果：¥27,925

计算明细：50克 × ¥558.50/克 × 99.9% = ¥27,925
```

**使用说明**：
- 此计算器仅供参考
- 实际回收价格需要专业检测后确定
- 包括成色鉴定、重量复核等环节

## 🚀 如何查看

### 方法一：从导航页开始（推荐）
```bash
# 在浏览器中打开导航页
open start.html
```
导航页包含所有页面的链接和功能说明，可以快速了解整个项目。

### 方法二：直接查看具体页面
```bash
open index.html          # 报价首页
open appointment.html    # 预约回收
open calculator.html     # 价格计算器
open profile.html        # 个人中心
```

### 方法三：本地服务器运行
```bash
# 进入prototype目录
cd prototype

# 启动本地服务器（Python 3）
python3 -m http.server 8000

# 或使用Node.js
npx http-server -p 8000

# 然后在浏览器访问
http://localhost:8000/start.html
```

### 页面切换
每个页面右上角都有固定的导航按钮，可以快速切换页面：
- 首页
- 预约
- 计算器
- 我的

## 💡 实现建议

### 数据更新
- **价格更新**：建议每日更新或实时同步国际金价
- **更新方式**：管理后台手动更新或API自动同步
- **更新时间**：显示在页面顶部，让用户知道价格时效性

### 预约管理
- **预约确认**：收到预约后30分钟内电话确认
- **短信提醒**：预约前1小时发送短信提醒
- **取消规则**：提前2小时可免费取消
- **记录保存**：所有预约记录需要保存在数据库

### 计算器逻辑
- **实时价格**：读取最新的回购价格
- **成色选项**：提供常见成色快捷选择（99.9%、99.5%、99%）
- **重量输入**：支持小数点输入，精确到小数点后2位
- **结果展示**：显示计算公式和明细
- **历史记录**：自动保存计算记录

### 用户体验
- **加载状态**：数据加载时显示骨架屏或Loading
- **错误提示**：网络错误、表单验证等友好提示
- **离线支持**：缓存最近一次的价格数据
- **分享功能**：支持分享给好友

## ⚠️ 注意事项

### 价格展示
1. 必须注明价格更新时间
2. 必须包含免责声明
3. 回购价不应高于销售价
4. 价格需要跟随市场行情合理调整

### 预约功能
1. 需要验证手机号格式
2. 预约时间不能选择过去的时间
3. 同一时间段预约数量限制
4. 需要记录预约来源和状态

### 计算器
1. 计算结果仅供参考
2. 必须说明以实物检测为准
3. 成色系数合理范围（90% - 100%）
4. 重量输入限制最大值

### 合规要求
1. 需要营业执照和经营许可
2. 客户身份证实名认证
3. 交易记录完整保存
4. 符合当地法律法规

## 🔧 技术实现

### 推荐技术栈
- **前端**：微信小程序原生 / uni-app
- **后端**：Node.js + Express / Java + Spring Boot
- **数据库**：MySQL + Redis
- **云服务**：腾讯云 / 阿里云

### 核心接口
```
GET  /api/prices              获取价格列表
POST /api/appointments        创建预约
GET  /api/appointments/:id    获取预约详情
PUT  /api/appointments/:id    更新预约
POST /api/calculate           计算价格
GET  /api/records             获取回收记录
POST /api/upload              上传图片（鉴定）
```
